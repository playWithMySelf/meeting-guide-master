# Meeting Master

会议概览总结生成工具，用于自动化生成会议汇报网页。

## 指令说明

本技能支持两个指令：

### 1. /meeting-guide

根据用户提供的会议资料自动生成会议概览总结网页。

**使用场景**：

- 用户传入包含 `audios` 和 `photos` 目录的文件夹路径
- audios 目录包含 `.md` 文件（会议纪要）和对应的 `.mp3` 文件
- photos 目录包含会议照片

**执行流程**：

1. **验证输入**
   - 检查路径是否存在
   - 检查是否包含 `audios` 和 `photos` 目录
   - 如不符合条件，提示用户重新传入
2. **遍历 MD 文件**
   - 遍历 `audios` 目录，找到所有 `.md` 文件
   - 每个 MD 文件必须包含 `# 速览` 和 `# 纪要` 两个章节
   - 如缺失章节，进行 AI 分析并**将补充内容写入原始 MD 文件**
   - 补充后需重新读取 MD 文件以获取最新内容
   - 如内容为空，忽略并提示用户
3. **选择主题模板**
   - **读取 `assets/themes/themes.json` 获取可用主题列表**
   - **如果只有一个主题**：直接使用默认主题
   - **如果有多个主题**：
     - 向用户展示所有可用主题列表，包含：主题名称、描述、风格、适用场景
     - 格式示例：
       ```
       可用主题列表：
       1. [默认主题] 简洁专业风格，蓝色系配色，适合正式会议、培训汇报
       2. [科技发布会] 科技感风格，深色模式，适合技术发布会、产品演示
       3. [商务简约] 极简商务风，黑白灰配色，适合商务洽谈、工作总结
       ```
     - 根据会议内容分析推荐最合适的主题（给出推荐理由）
     - 等待用户确认或选择其他主题
4. **生成结果目录**
   - 在 audios 同级目录创建 `result` 文件夹
   - 将选中的主题模板文件（`data.js` 和 `template.html`）拷贝至 result 目录，重命名`template.html`为`index.html`
5. **填充数据**
   - 遍历 `audios` 和 `photos` 目录
   - 按照真实数据填充 `data.js`
   - 在 data.js 中设置正确的文件路径（指向同级的 audios 和 photos 目录）
   - 没有识别到数据时设置为空数组，并给用户提示

**数据生成规则**：

- **会议议题列表**：优先级为 MD 文件标题 > 文件名
- **核心总结**：AI 汇总所有 MD 中的重要观点，每条不超过 40 字，每个 MD 最多 2 条
- **演讲嘉宾**：从纪要中提取发言人信息
- **时间线**：每个录音对应一个时间线项目
- **发言人观点**：提取重要发言人的核心观点

### 2. /meeting-new-template

根据用户场景和要求，基于默认主题生成新的主题。

**使用场景**：

- 用户需要自定义会议概览总结的外观和样式
- 用户指定了特定的场景（如培训、研讨会、发布会等）

**执行流程**：

1. **分析需求**
   - 了解用户的场景和要求
   - 确定主题的风格、配色、布局等
2. **基于默认主题修改**
   - 复制 `assets/themes/default` 中的模板文件（`data.js` 和 `template.html`）
   - 根据需求修改 `template.html` 的样式和结构，优先使用/ui-ux-pro-max skill进行template.html的修改，注意页面背景颜色与内容是否协调
   - 保持 `data.js` 的数据结构不变
3. **保存新主题**
   - 在 `assets/themes` 目录下创建新主题文件夹（使用英文小写，如 `tech-conference`）
   - 保存修改后的模板文件
4. **注册主题**
   - **读取 `assets/themes/themes.json` 文件**
   - **追加新主题信息到 themes 数组**，包括：
     - `id`: 主题唯一标识（与文件夹名一致）
     - `name`: 主题显示名称
     - `description`: 主题描述（50字以内）
     - `style`: 风格特点（如：科技感、商务风、简约风）
     - `colorScheme`: 配色方案（如：蓝色系、暖色系、深色模式）
     - `suitableFor`: 适用场景数组（如：["发布会", "技术分享"]）
     - `path`: 主题文件夹路径
     - `createdAt`: 创建日期
     - `isDefault`: false
   - **保存更新后的 themes.json**
5. **完成提示**
   - 提示新主题位置
   - 列出当前所有可用主题供用户查看

## 目录结构

```
项目目录/
├── assets/
│   └── themes/
│       ├── themes.json       # 主题注册表，记录所有可用主题信息
│       ├── default/          # 默认主题
│       │   ├── data.js       # 数据文件模板
│       │   └── template.html # HTML模板
│       └── {其他主题}/        # 用户创建的自定义主题
│           ├── data.js
│           └── template.html
├── meetings/
│   └── demo/                 # 用户数据示例
│       ├── audios/           # 录音和纪要
│       │   ├── 1-议题1.md
│       │   ├── 1-议题1.mp3
│       │   └── ...
│       ├── photos/           # 会议照片
│       │   └── ...
│       └── result/           # 生成的结果
│           ├── data.js       # 填充后的数据文件
│           └── index.html    # HTML页面（由template.html复制重命名）
```

## 资源引用

### references/data-structure.md

data.js 数据结构的详细说明，包含所有数据字段的定义和生成规则。

**读取时机**：当需要生成或修改 data.js 文件时读取。

### references/md-format.md

MD 文件格式规范，说明 audios 目录中 MD 文件的必需章节和格式要求。

**读取时机**：当解析 MD 文件或补充缺失章节时读取。

### assets/themes/themes.json

主题注册表，记录所有可用主题的信息。

**数据结构**：
```json
{
  "version": "1.0",
  "lastUpdated": "2026-04-02",
  "themes": [
    {
      "id": "主题唯一标识",
      "name": "主题显示名称",
      "description": "主题描述",
      "style": "风格特点",
      "colorScheme": "配色方案",
      "suitableFor": ["适用场景1", "适用场景2"],
      "path": "主题文件夹路径",
      "createdAt": "创建日期",
      "isDefault": true/false
    }
  ]
}
```

**读取时机**：
- `/meeting-guide` 指令：获取可用主题列表供用户选择
- `/meeting-new-template` 指令：读取并追加新主题信息

### assets/themes/default/

默认主题模板文件：

- `data.js` - 数据文件模板
- `template.html` - HTML 页面模板

**使用时机**：

- `/meeting-guide` 指令：复制到 result 目录并填充数据
- `/meeting-new-template` 指令：作为新主题的基础模板

## 注意事项

1. **MD 文件格式**：确保每个 MD 文件包含 `# 速览` 和 `# 纪要` 两个一级标题
2. **文件对应关系**：MD 文件应与 MP3 文件同名（仅扩展名不同）
3. **数据完整性**：没有识别到的数据设置为空数组，并在任务执行后提示用户
4. **核心总结长度**：每条总结不超过 40 字，确保简洁有力
5. **议题来源优先级**：优先使用 MD 文件中的标题，其次使用文件名
