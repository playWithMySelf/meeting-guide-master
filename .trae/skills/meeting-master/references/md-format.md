# MD 文件格式规范

本文档说明 audios 目录中 MD 文件的格式要求。

## 文件命名

MD 文件应与对应的 MP3 文件同名，仅扩展名不同：
- 音频文件：`1-会议内容.mp3`
- MD 文件：`1-会议内容.md`

## 必需章节

每个 MD 文件必须包含以下两个章节：

### 1. 速览

包含 MP3 文件的完整转写内容，格式如下：

```markdown
# 速览

00:00

第一段转写内容...

00:45

第二段转写内容...
```

### 2. 纪要

包含对转写内容的摘要和整理，格式如下：

```markdown
# 纪要

## 章节标题

- **要点1**：详细说明
- **要点2**：详细说明

## 另一个章节

- **要点3**：详细说明
```

## 完整示例

```markdown
# 速览

00:00

河南，地处中原腹地，是中华文明的重要发祥地...

00:45

首先推荐的是世界文化遗产——龙门石窟...

# 纪要

## 世界文化遗产类

- **龙门石窟**：位于洛阳市，开凿于北魏至宋代...

## 历史文化类

- **少林寺**：位于登封市嵩山...
```

## 缺失章节处理

当 MD 文件缺失章节时：

1. **缺失"速览"章节**：
   - 如果存在对应的 MP3 文件，提示用户需要转写
   - 如果无法获取转写内容，设置为空字符串

2. **缺失"纪要"章节**：
   - 使用 AI 分析"速览"内容生成纪要
   - 按照标准格式补充纪要章节

3. **内容为空**：
   - 忽略该文件并提示用户
